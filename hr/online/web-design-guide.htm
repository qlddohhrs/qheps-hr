HR Web design guide

Introduction

For the purposes of creating a consistent look and feel to the QHEPS HR website and beyond, the following collection of styles and associated code templates has been created. As long as the required stylesheet and scripts are referenced within your website's content framework (header, footer, etc.), all of the elements below will be available. Utilising them for the prescribed purposes helps create a unified and logical experience for our site visitors by being able to:

  • visually distinguish what type of content can be expected where
  • provide various ways to drawn attention to the content that matters most.

As with anything, overuse can have diminishing returns.

Some of these elements work best on their own, however most have been designed in a modular fashion allowing combinations of them where appropriate.

  • For SMEs and content creators, being aware of the possibilities below should at least give you a starting point on how your content can be organised and presented.
  • For Publishers a basic understanding of HTML & CSS is expected. Adapting these individually to your content, or mix and matching these various building blocks should hopefully be straightforward.

Images

Full-width

If you have an image that you want to ensure is stretched to the full width of it's container whether that be the entire body area of your page or inside any other box or container, just add the class full-width.

Code example:
<img src="PathToImage" alt="AltDescriptionHere" class="full-width"/>

Magnify

If you have an image that needs to be viewed large to see all the detail (charts, diagrams, etc.) but would prefer it didn't take up a lot of room on your page, you can use the magnify class to have it enlarge to twice the size when the cursor hovers over it. Just set the height and width at half the size of the original image size.

Code example:
<img height="200" width="200" src="PathToImage" alt="AltDescriptionHere" class="magnify"/>

Font Awesome icons

Icon fonts provide a quick and easy way to incorporate scalable vector images and adap them to your content's style. We have chosen the license-free Font Awesome font and CSS toolkit which is very commonly used and included by default in popular frameworks such as Bootstrap.

Anatomy of a Font Awesome icon tag

The structure of a typical icon tag is comprised of:

  • Opening and closing tags.
  • Class of far which is mandatory to indicate a Font Awesome icon. Options include:
    • fas = solid weight
    • far = regular weight
    • fal = light weight [Coming soon]
    • fab = specified when it's a brand icon (The search index below will tell you).
  • fa-li is only used if you are placing within a list.
  • fa-globe is the all important name class that represents your icon choice.
  • fa-lg represents the size. Options include:
    • fa-xs = x-small
    • fa-sm = small
    • fa-lg = large (This is usually the smallest we use).
    • fa-2x to fa-10x = 2em to 10em size.

Icon index

If you are replacing an existing icon, the only change is the class that includes the icon name, eg. fa-dollar, fa-calendar.

Here is a list of most commonly used ones.

  • Bullet Point (fa-circle) – only icon with a size of fa-sm
  • Chain link (fa-link) – internal links
  • Globe (fa-globe) – external links
  • Text Doc (fa-file-text) – Word documents
  • PDF Doc (fa-file-pdf-o) – PDF documents
  • Contact (fa-address-book)
  • Email (fa-envelope)
  • Calendar Event Item fa-calendar-check-o)
  • Calendar (fa-calendar)
  • Location/Map Marker (fa-map-marker)
  • Dollar Sign (fa-dollar)
  • Warning_Safety (fa-warning)
  • User (fa-user)
  • Phone (fa-phone)
  • FAQ (fa-question-circle)

Asides (right-column)

An aside in the right-column sidebar is put together primarily from one right_column-sectionName.inc file. If you have more than one aside you will likely have some repetitive components across all your pages. For best practice, these should be referenced as separate includes (in the event they need to be updated in future.)

Common ones would be:

  • Logo/Branding
  • Branch-customised search widget

While others are unique to your particular section:

  • Supporting resources – any documents that are either linked throughout the page body or provided as additional material.
  • Related links – internal/external pages outside of the section nav.
  • Contact info – tailored help direction for the section's material.

Building an aside

Start by:

  • creating an includes folder at the top-level of your site section's folder.
  • Upload the 2 files detailed below: right_column-sectionName.inc and quick_links-sectionName.inc.
  • Any global ones for branding, search, etc. hopefully are already in place, otherwise they would need to be placed in the root /include folder of your branch site.

Branch search include

To customise the search, the form action needs to be adjusted with a comma-separated list of the site sections that searches should be narrowed to.

For example: action="javascript: fbsearch('s-search','qheps.health.qld.gov.au/hr');"

Code example:
<div class="sidebar_section">
    <h3 class="sidebar">Search BranchName</h3>
    <div class="hr-form hr-search">
        <script type="text/javascript" src="http://qheps.health.qld.gov.au/_includes/scripts/funnelback.js"></script>
        <form action="javascript: fbsearch('s-search','qheps.health.qld.gov.au/');" name="s-search">
            <input name="query" class="query" size="25" type="text" placeholder="Type search terms + enter" />
            <input id="searchbut" type="submit" value="" />
        </form>
    </div>
</div>

Responsive grids

A grid framework from 2 to 8 columns is available when you wish to display columns of content that will be evenly distributed horizontally within the main body of your page. The responsive element to this ensures that as the page decreases in width, the columns reduce down gradually until the content is stacked vertically in one column.

This grid is used extensively for navigation button layouts but any combination of images, buttons, text or other media can be laid out in this fashion. If an image is the only content in a particular column, set it's width to 100% as it will then scale/expand down responsively with the column. Consequently you should produce an image that is as wide as the maximum width may be.

Buttons

These are modeled off the commonplace Bootstrap-style buttons (officially used on Internet site for consistency). Typically they would be used to style an a anchor element, but technically the class can be applied to any container. If buttons are included in an aside, it's recommended to wrap in a p tag for better spacing.

Standard button (btn)

The button size automatically expands to fit the content. Any content placed inside a button will center vertically. Regardless of the following variations used, the btn class is always required.

Click me!

Code example:
<a href="ButtonDestinationURL" class="btn">ButtonText</a>

Large button (btn-lg)

Same as standard button with larger text size. With any style you can include a fontawesome icon, and prepend it using the class pull-left or append it with pull-right.

Attention

Code example:
<a href="ButtonDestinationURL" class="btn btn-lg"><i class="fa fa-star fa-lg pull-left"></i>ButtonText</a>

Nav button (btn-nav)

The nav button style is similar to a large button with a fixed width to suit a 2, 3 or 4 across grid layout. This example further incorporates an image which are automatically floated left (height & width must be specified), and a caret affixed to the far right using a pull-right class.

Font awesome icon small
Font awesome icon large
No images
Custom graphic
Code example:
<a href="ButtonDestinationURL" class="btn btn-nav bg-off-white black-text"><img src="" alt="icon" height="40" width="40">ButtonText<i class="fa fa-caret-right fa-lg pull-right"></i>

Coloured buttons (bg-colorname)

You can custom colour any button (or any block element for that matter). Pick a background color from the Colour palette and replace the bg-alert-red class used in the example. Custom campaign colours should be added to the stylesheet as needed so that the hover state is applied correctly.

Note the addition of class black-text if the button is too light to have white text.

Custom colour

Code example:
<a href="ButtonDestinationURL" class="btn btn-lg bg-alert-red"><i class="fa fa-warning fa-lg pull-left"></i>ButtonText</a>

Slideshows/Carousels

A simple and responsive carousel slider is available beyond the previous QHEPS version. See the QHEPS HR homepage for an example.

You will need:

  • Your images uploaded in an appropriate folder.
  • a new include file that lists all the slides (image path, title, description text)
  • an include line inserted on the page where the slideshow is to be displayed.

The slideshow will use our default settings for speed, transition and random display. If you wish to customise this and know how to insert javascript within your page you can refer to the Bx-slider options page for more info.

Include file

Customise the following template with your slide details and save with an appropriate name usually to your top-level /include folder.

We suggest a minimum of 3 and maximum of 5 slides at any one time. Given the time it's displayed and that visitors typically spend looking at these, your text should be large and descriptions as brief as possible.

Code example:
      <ul class="bxslider">
    <li>
        <a href="DestinationURLWhenImageIsClicked">
            <img src="PathToImage" alt="AltTextForImage" /></a>
        <p class="title">Title</p>
        <p>ShortDescription</p>
    </li>
  </ul>

Abbreviations

To meet accessibility requirements the first occurance of any Acronym on a page must be accompanied by the full name. Commonly this might be done in parentheses. If this seems overkill for commonly known ones, the minimum to meet requirements would be to use the ABBR tag.

Code example:
<abbr title="Department of Health">DoH</abbr>

Footnotes

This footnote solution is suitable for citations or simply some additional aside information rather than breaking the flow.Rather than the paper-bases paradigm of jumping to the bottom of the page and back, these footnotes will pop-up in a box to the right when the  is hovered over.
Code example:
<sup><a href="#" class="fn"><em class="fa fa-question-circle fa-lg"></em><span>InsertFootnoteHere</span></a></sup>

Miscellaneous

Figure Caption

Often it is appropriate to include some caption text underneath an image, chart, video, etc. There is a dedicated HTML5 tag for this call figcaption. TeamSite however does not recognise standardised HTML5 so in lieu of this just add a class by the same name to your paragraph tag.

Code example:
<p class="figcaption">CaptionText</p>

Hiding element

At times you may wish to temporarily hide a section of your page instead of deleting. For instance, either it has a fixed campaign length but will likely be re-used next year or a page has been approved to publish except for parts of it that aren't yet.

Simply add the hide class to the opening tag of the element or entire section you wish to hide on the page.

Code example:
<div class="hide">ContentToHide</div>

Responsive embedded videos

Sometimes we're asked to add a video to a page and you receive some embed code from the client or Production for a Vimeo or YouTube video. If you wrap the provided code in another div and give it the class video-container, it will automatically make the video responsive to the container's width, but with with a maximum width of 768px so it doesn't overwhelm the page on large displays. This could be the entire body section, part of a responsive grid layout, or any sized container.

Code example:
<div class="video-container"><iframe width="560" height="315" src="https://www.youtube.com/embed/YBK_1WrM294?rel=0" frameborder="0" allowfullscreen></iframe></div>

Referencing from your TeamSite page

As an example for any code like this that TeamSite doesn't comprehend, it must be saved externally as in XXX.inc file and linked as such.

Code example:
<!--#include virtual="/pathtofolder/video-example.inc" -->
